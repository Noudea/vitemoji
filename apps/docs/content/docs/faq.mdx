title: FAQ
description: Common questions about what vitemoji transforms and how matching works.
---

## Does it rewrite plain strings everywhere?

No. `vitemoji` only rewrites text that is rendered in JSX:

- JSX text nodes
- string literals inside JSX expression containers
- template literal text inside JSX expression containers

It does not rewrite ordinary strings elsewhere in your code.

## Should I use `emojifyText()`, `useEmojifier()`, or `useCreateEmojifier()`?

Use `emojifyText()` when you want a synchronous helper for:

- Node or server-side code
- scripts and tooling
- simple string conversion without runtime loading

Use `useEmojifier()` in React apps when you want:

- browser/runtime usage
- locale-aware loading in the client
- full locale and shortcode preset control
- one shared config for an app or subtree

Use `useCreateEmojifier()` when only one screen or one component needs runtime
loading and you do not want to add a provider.

`VitemojiProvider` + `useEmojifier()` is the recommended React runtime API.

## Does it transform JavaScript files?

Only files that match the `include` pattern are transformed. The default pattern
is:

```ts
/\.[jt]sx$/
```

That means the default setup targets `.jsx` and `.tsx` files.

## Does it touch dependencies?

No. Files inside `node_modules` are skipped.

## What happens if a file cannot be parsed?

The plugin returns the original source unchanged.

## How are conflicts resolved?

These rules apply:

- the first matching shortcode claim wins when multiple shortcode datasets define
  the same token
- the longest phrase wins within a match category
- category priority is shortcodes, hexcodes, names, then keywords

## Why does React runtime usage load asynchronously?

Runtime React usage loads generated locale and shortcode chunks on demand.

That keeps the initial browser bundle smaller than loading every locale and every
shortcode dataset up front. `useCreateEmojifier()` returns `isReady` so a local
component can show a loading state, while `VitemojiProvider` can handle loading
for a whole subtree.

## Can I mix locales and shortcode presets?

Yes, but your selected shortcode presets must support the locales you request. If
none of the requested presets support one of your locales, the plugin throws a
setup error so the mismatch is visible immediately.

## Why do locale and shortcode preset combinations sometimes throw?

`vitemoji` validates that each requested locale is supported by at least one of
your requested shortcode presets.

For example:

- `locales: ["bn"]` with `shortcodePresets: ["github"]` throws
- `locales: ["bn"]` with `shortcodePresets: ["cldr"]` works

This is intentional. The package fails fast instead of silently falling back to a
different shortcode dataset.
